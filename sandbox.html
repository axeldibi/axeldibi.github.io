Exécution sécurisée avec Windows Sandbox

Pour ouvrir en toute sécurité un fichier contenant des macros potentiellement malveillantes, l’idée est de l’isoler du système hôte afin que, même si la macro exécute du code dangereux, elle ne puisse pas toucher vos données ni votre réseau.

Utiliser Windows Sandbox (Windows 10/11 Pro et Enterprise)
Windows Sandbox est un mini-Windows jetable qui se détruit à la fermeture. Si vous n’avez pas Windows Sandbox, vous pouvez utiliser une machine virtuelle avec un système d’exploitation isolé, comme VirtualBox ou VMware. Cependant, Windows Sandbox est plus simple à configurer et à utiliser pour ce type de tâche.

Activez la fonctionnalité :

  • Ouvrez Panneau de configuration → Programmes → Activer ou désactiver des fonctionnalités Windows.
  • Cochez Windows Sandbox (ou bac à sable Windows), cliquez sur OK et redémarrez.

Lancez Windows Sandbox :
Tapez « Sandbox » dans le menu Démarrer et ouvrez l’application.

Importez et ouvrez votre fichier :

  • Dans la fenêtre Sandbox, faites glisser (ou copier/couper coller) votre fichier contenant les macros depuis l’explorateur de l’hôte vers celui de la Sandbox.
  • Coupez l’accès réseau de la Sandbox. Pour cela, vous pouvez exécuter dans le PowerShell de la Sandbox: 
     Disable-NetAdapter -Name "Ethernet" -Confirm:$false
  • Ouvrez le fichier avec l'application dédiée (par exemple Excel pour les .xlsm), Installée dans la Sandbox .

Travaillez en toute tranquillité :
À la fermeture de la Sandbox, tout le contenu (y compris le fichier et les éventuelles infections) est définitivement supprimé.

Pour en savoir plus, visitez https://learn.microsoft.com/fr-fr/.../windows-sandbox-configure-using-wsb-file .

Mettre en application ces principes de sécurité :

  • Projet Excel (version Excel avec macros)
  • Projet Access (version Access avec macros)
